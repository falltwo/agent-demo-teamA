# Project Flow Status

## 目前總進度

目前專案主軸已從「合約生成與編輯」往下推進到「知識庫管理」。

整體狀態可概括為：

- 文件輸入與解析：已完成第一版，可上傳並解析 `txt` / `md` / `pdf` / `docx`
- 合約生成與編輯：已完成第一版，可生成、編修與 Redline
- 審批工作流：已完成單一使用者版本
- 知識庫管理：進行中，為目前主要開發重點
- 解釋與報告：尚未完整展開
- 安全與合規：尚未完整展開

## 依流程圖的目前狀態

### 1. 文件輸入與解析

已完成的項目：

- PDF / DOCX 解析
- OCR 掃描合約
- 多版本差異比對
- 文件灌入向量庫

目前這一層已可支撐後續 RAG 與合約審閱流程。

### 2. 合約生成與編輯

已完成的項目：

- 合約模板與草稿生成
- 條款編修
- Redline 顯示

已補上的規格方向：

- 用 RAG 限制條款生成範圍
- 法條編號存在性驗證
- 金額、天數、比例合理性檢查
- 每條條款顯示來源條款 ID 與信心分數

這一層已有規格與部分流程，但條款級來源追蹤仍待進一步完整落地。

### 3. 審批工作流

目前採單一使用者模式，不走跨部門審批。

已完成的項目：

- 法務審閱 / 法律專家單步驟流程
- 送簽前的審閱狀態
- 電子簽署銜接區塊
- 到期提醒與義務追蹤區塊
- 新手導向的文案簡化

已調整方向：

- 移除多方協作審批
- 聚焦個人使用情境
- AI 檢閱需鎖 RAG，不只靠規則檢查

### 4. 知識庫管理

這是目前主要開發重點。

已完成的項目：

- 法條 / 判例 dataset 保存結構
- 資料正規化與 `content_hash` 比對
- 同步 job 紀錄
- CLI 管理入口
- provider / adaptor registry
- seed provider
- 接回既有 `rag_ingest.py` 流程

目前已建立的後端模組：

- `knowledge_base_jobs.py`
- `knowledge_base_sync.py`
- `knowledge_base_providers.py`
- `tools/knowledge_base_admin.py`

目前已支援的後端操作：

- 同步 seed 資料
- 同步外部 JSON 檔
- 寫入既有 RAG 索引
- 查 dataset 狀態
- 查同步工作紀錄
- 列出 provider
- 執行 provider 同步

已確認的產品方向：

- 知識庫管理不放在主畫面
- 僅保留後端 / CLI / 後續 admin 入口

### 5. 解釋與報告

目前只有部分概念與相鄰功能，尚未完整展開：

- 條款白話解釋
- 風險評分報告
- 合約摘要生成

### 6. 安全與合規

目前尚未完整展開：

- 角色權限控管
- 稽核日誌（Audit Log）
- 人工覆審機制

## 當前重點

目前最主要的工作區塊是「知識庫管理」。

原因是：

- RAG 品質取決於法條與判例資料是否完整、可更新、可追蹤
- 後續 AI 檢閱、法條驗證、風險分析都依賴這一層
- 現在已完成基礎框架，適合往正式資料來源與排程更新推進

## 下一步

知識庫管理接下來的重點如下：

1. 接正式法條來源 adaptor
2. 接正式判例 / 裁判書來源 adaptor
3. 補強同步失敗與成功的 job 記錄
4. 設計定期更新與索引更新策略
5. 讓後續 AI 檢閱可直接依賴更新後的法規 / 判例知識庫
