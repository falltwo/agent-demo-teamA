# 合約／採購法遵審閱助理 — Demo 操作指南

> 供現場 Demo、3 分鐘競賽影片或決賽簡報使用。依此流程可完整展示：上傳合約 → 灌入 → 一鍵審閱（僅知識庫／含法條）→ 風險列表與檢索片段。

---

## 一、事前準備（Demo 前一日或當天早上）

### 1.1 環境與 API

| 項目 | 作法 |
|------|------|
| **.env** | 複製 `.env.example` 為 `.env`，填入 **PINECONE_API_KEY**、**PINECONE_INDEX**、**GOOGLE_API_KEY**。 |
| **法條查詢** | 若要示範「合約＋法條查詢」，請填 **TAVILY_API_KEY**（司法院／網路搜尋）。 |
| **Pinecone** | 確認 index 已建立、維度與 `GEMINI_EMBED_MODEL` 一致（如 768）。 |

### 1.2 建議開啟的 .env（合約審閱體驗更好）

```env
RAG_USE_HISTORY_FOR_QUERY=1
RAG_MAX_HISTORY_TURNS=12
# 可選：多查詢、BM25 Hybrid
# RAG_MULTI_QUERY=1
# RAG_USE_BM25=1
```

### 1.3 範例合約與測試資料從哪來

- **專案內建**：`data/sample_contract_NDA.txt` 為簡短保密協議範本（條款、違約金、管轄等），可直接用來測合約審閱。
- **自己放**：將 1～2 份合約放到 `data/`（.txt / .md / .pdf / .docx；PDF 建議文字型，避免掃描版）。
- **公開範本**：可搜尋「政府採購契約範本」「NDA 範本」「租賃契約範本」等，下載文字版或複製條文存成 .txt / .md 放入 `data/`。
- **政府採購**：行政院公共工程委員會「政府採購契約範本」、各機關招標文件中的契約條款，可節錄存檔測試。
- **或現場上傳**：Demo 當天在 Streamlit「上傳並灌入文件」選檔案即可，不需事先放 data/。

### 1.4 先跑一次灌入（若用 data/）

```bash
uv run rag_ingest.py
```

看到「已灌入完成：N chunks」即可。若只用「上傳＋灌入」流程可略過。

## 二、啟動 Demo

在專案根目錄執行：

```bash
uv run streamlit run streamlit_app.py
```

瀏覽器會開啟（預設 http://localhost:8501）。若需指定 port：

```bash
uv run streamlit run streamlit_app.py --server.port 8502
```

## 三、建議 Demo 流程（約 2～3 分鐘）

### 情境 A：已有 data/ 且已灌入

1. **開場**（約 20 秒）  
   「這是合約／採購法遵審閱助理，可以上傳合約後用自然語言問風險、條款，並可選查司法院法條。」

2. **列出知識庫**（約 15 秒）  
   輸入：「列出目前知識庫有哪些文件」→ 應走 `list_sources`，展示已灌入的合約／文件。

3. **一鍵審閱（僅知識庫）**（約 40 秒）  
   - 側欄展開「合約審閱提示」→ 點 **「一鍵審閱（僅知識庫）」**。  
   - 或直接輸入：「請審閱這份合約的風險條款」。  
   - 展示：條款類型、風險等級、建議、**檢索片段可展開對照**。

4. **合約＋法條查詢**（約 50 秒，需 TAVILY_API_KEY）  
   輸入：「幫我做合約風險評估並查相關法條」或點 **「一鍵審閱（含法條查詢）」**。  
   展示：風險摘要、**相關法條重點**、法條字號清單、來源、文末免責聲明。

5. **收尾**（約 15 秒）  
   「第一輪審閱可由約 1 小時縮短到約 10～15 分鐘，且可追溯出處、可選查法條。」

---

### 情境 B：從零開始（上傳 → 灌入 → 審閱）

1. **開場**（同上）

2. **上傳合約**（約 30 秒）  
   展開「為此對話上傳並灌入文件」→ 選一份 .pdf / .txt / .md 合約 → 按 **「灌入到向量庫」**，等出現「已灌入 N 個 chunks」。

3. **一鍵審閱**（約 40 秒）  
   側欄點「一鍵審閱（僅知識庫）」或輸入「請審閱這份合約的風險條款」。

4. **含法條**（約 50 秒，需 TAVILY_API_KEY）  
   輸入「合約風險評估並查相關法條」或點「一鍵審閱（含法條查詢）」。

5. **收尾**（同上）

## 四、Demo 時要注意的點

| 要點 | 說明 |
|------|------|
| **嚴格模式** | 側欄「嚴格只根據知識庫回答」預設為**不勾選**，合約專用工具才會被路由到。 |
| **一鍵審閱** | 在側欄「合約審閱提示」內，有「僅知識庫」與「含法條查詢」兩顆按鈕，可固定走對應 tool。 |
| **檢索片段** | 回答下方「查看檢索片段」可展開，展示引用來源，方便說明**可追溯**。 |
| **網路** | 現場若有防火牆，先確認可連 Pinecone、Google（Gemini）、Tavily（若用法條查詢）。 |

---

## 五、若現場不穩：預錄影片備援

1. 依「三、建議 Demo 流程」預錄一段 **2～3 分鐘** 操作影片（含上傳→灌入→一鍵審閱→檢索片段）。  
2. 存成檔案，決賽時若網路或 API 異常可改播影片。  
3. 競賽 3 分鐘影片規定：YouTube 不公開、控制在 3 分鐘內，可與此流程一致。

## 六、快速檢查清單（Demo 前 5 分鐘）

- [ ] `.env` 已填 PINECONE_API_KEY、PINECONE_INDEX、GOOGLE_API_KEY  
- [ ] 要示範法條查詢時已填 TAVILY_API_KEY  
- [ ] 已執行 `uv run rag_ingest.py`（若用 data/）或準備好要上傳的合約  
- [ ] `uv run streamlit run streamlit_app.py` 可正常開啟  
- [ ] 側欄「嚴格只根據知識庫回答」**未勾選**  
- [ ] 能成功問「請審閱這份合約的風險條款」並看到合約專家回答與檢索片段
